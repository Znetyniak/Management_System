Theory & Defence Notes for Fleet Management System

====================================================

1. Project Overview

Своїм курсовим проєктом я розробив інформаційну систему Fleet Management System (варіант 7), призначену для керування автопарком, водіями, поїздками, технічним обслуговуванням та витратами.

Система побудована на основі трирівневої архітектури:
Presentation → Service Layer → Data / Repository Layer.

Моя мета — створити повноцінний веб-додаток із зручною адміністративною частиною, користувацьким інтерфейсом та модулем звітності.

2. Використані технології

У проєкті я використав:

Python 3.13 — мова реалізації

Flask 3 — веб-фреймворк

Flask-SQLAlchemy — ORM

Flask-Migrate — міграції

SQLite3 — база даних fleet_management.db

Jinja2 — шаблонізатор HTML

Custom CSS (без Bootstrap) — оформлення UI

openpyxl — створення Excel-звітів

3. Шаблони проектування (GoF)

У межах курсового я впровадив кілька шаблонів проєктування:

✔ Singleton

Використовується для створення єдиного екземпляра об’єкта бази даних (db).

✔ Repository

Повністю ізолює SQL-операції від бізнес-логіки.
У мене є окремі репозиторії під кожну сутність:
vehicle_repository.py, driver_repository.py, trip_repository.py, expense_repository.py, maintenance_repository.py.

✔ Service Layer (аналог Facade)

Service-класи інкапсулюють бізнес-логіку та взаємодію між репозиторіями.
Наприклад:
TripService, ExpenseService, MaintenanceService, VehicleService, ReportService.

✔ Factory Method

Використовується для уніфікованого створення Excel-звітів різних типів (Usage, Expenses, Maintenance).

✔ Strategy

Реалізовано у звітному модулі — кожен звіт має власний алгоритм формування.

✔ Observer (опційно)

Логіка нагадувань про майбутнє техобслуговування (може бути розширена).

4. Детальний опис файлів
Основні файли

run.py — головна точка запуску

app/init.py — фабрика застосунку create_app()

config.py — налаштування БД (включно з реальним шляхом до SQLite)

extensions.py — ініціалізація ORM і міграцій

Models

vehicle.py — бренд, модель, рік, пробіг, VIN

driver.py — особисті дані, номер посвідчення

trip.py — маршрут, кілометраж, статус

expense.py — фінансові витрати автопарку

maintenance.py — ТО: дата, тип, наступна дата

user.py — користувачі, ролі (admin/user)

Repositories

CRUD-операції та специфічні запити:

пошук авто

активні поїздки

витрати за період

майбутні ТО
Вони повністю відокремлюють бізнес-логіку від SQL.

Services

VehicleService — керування автопарком

DriverService — водії

TripService — запуск і завершення поїздок

ExpenseService — облік витрат

MaintenanceService — ТО

ReportService — три типи звітів: Usage, Expenses, Maintenance

Усі контролери звертаються тільки до сервісів — це дозволяє підтримувати SOLID (особливо SRP та OCP).

Controllers

auth_controller.py — логін, signup, менеджмент сесій

store_controller.py — основний інтерфейс користувача

admin_controller.py — CRUD-доступ для admin

звітні контролери — експорт у Excel та фільтрація

Templates

base.html — єдиний макет

_/includes — navbar, модальні вікна, flash

Store/ — dashboard, списки, деталі

Admin/ — панель керування сутностями

звітні сторінки — фільтри + таблиці + картки

Static

static/css/style.css — кастомний повністю переписаний стиль з картками, таблицями, тінями, адаптивністю та оновленим navbar / footer.
